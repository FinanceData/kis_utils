# KIS Utils Agent Skill

한국투자증권(KIS) API를 활용하여 국내 주식 시장의 데이터를 조회하고 거래를 수행하는 고성능 금융 에이전트 스킬입니다.

## 🚀 Overview
이 스킬은 실시간 주식 시세, 일일 시세 데이터, 계좌 잔고 조회 및 주식 주문(매수/매도) 기능을 제공합니다. LLM은 이 스킬을 사용하여 사용자의 투자 관련 질문에 답하거나 직접적인 거래 요청을 처리할 수 있습니다.

## 🛠 Features
- **실시간 데이터**: 현재가, 전일 대비 등락, 거래량 등 실시간 마켓 데이터 조회.
- **차트 데이터**: 일/주/월/년 단위의 역사적 시세 데이터 조회.
- **자산 관리**: 계좌 잔고, 예수금, 보유 종목 및 수익률 실시간 확인.
- **스마트 주문**: 매수 가능 수량 계산 및 시장가/지정가 주문 수행.

## 📖 Tool Details

### 1. 시세 및 데이터 (Market Data)
- **`price(stock_code: str)`**: 종목의 현재 가격과 상세 상태를 가져옵니다.
  - *사용 시점*: "현재 얼마야?", "상승 중이야?", "거래량 어때?" 등의 질문 시.
- **`daily(stock_code: str, period: str = "D", ...)`**: 과거 시세 데이터를 조회합니다.
  - *사용 시점*: "최근 흐름 보여줘", "어제 종가 얼마였어?", "주봉 데이터 확인해줘" 등.

### 2. 계좌 및 관리 (Account Management)
- **`balance()`**: 전체 계좌 상태를 확인합니다.
  - *사용 시점*: "내 잔액 얼마야?", "지금 뭐 가지고 있어?", "수익률 알려줘" 등.
- **`buyable(stock_code: str, price_val: int)`**: 주문 전 가능 수량을 확인합니다.
  - *사용 시점*: "삼성전자 몇 주나 살 수 있어?", "100만원으로 이거 얼마나 사?" 등.
- **`order_history(start_date: str, ...)`**: 과거 거래 내역을 조회합니다.
  - *사용 시점*: "어제 뭐 샀어?", "체결 됐어?" 등.

### 3. 거래 (Trading)
- **`buy(stock_code: str, quantity: int, price: int, ...)`**: 주식을 매수합니다.
  - *사용 시점*: "매수해줘", "사줘" 등 직접적인 매수 요청 시.
- **`sell(stock_code: str, quantity: int, price: int, ...)`**: 주식을 매도합니다.
  - *사용 시점*: "팔아줘", "익절해줘" 등 직접적인 매도 요청 시.

## 💡 Usage Guidelines for LLM
1. **종목 코드 확인**: 사용자가 종목명만 말한 경우, 내부 지식을 활용하거나 검색을 통해 6자리 종목코드를 먼저 파악한 후 도구를 호출하세요.
2. **주문 전 검증**: 매수 주문(`buy`) 전에는 반드시 `buyable`을 호출하여 가용 자금을 확인하고, 매도 주문(`sell`) 전에는 `balance`를 통해 보유 수량을 확인하세요.
3. **데이터 처리**: 모든 반환 데이터는 딕셔너리 형태입니다. 숫자가 문자열로 반환될 수 있으므로 필요시 숫자형으로 변환하여 계산하세요.
4. **안전 제일**: 주문과 같은 자산 이동 작업 시에는 사용자에게 "삼성전자 1주를 70,000원에 매수할까요?"와 같이 명확하게 재확인 루프를 만드세요.

## 📝 Examples
- **User**: "삼성전자 지금 얼마야?"
  - **Agent**: `price("005930")` 호출
- **User**: "내 계좌에서 수익률 제일 좋은 거 뭐야?"
  - **Agent**: `balance()` 호출 후 `evlu_pfls_rt` 값 비교 분석
- **User**: "나 지금 돈 얼마 있어? 삼성전자 최대로 사줘."
  - **Agent**: `balance()`로 예수금 확인 -> `buyable()`로 최대 수량 확인 -> `buy()`로 주문 수행
